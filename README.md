# 왕립 계승 아카데미 봇

마스토돈 기반의 프린스/프린세스 육성 커뮤니티 자동봇입니다.
플레이어는 매 턴 행동을 선택해 수치를 성장시키고, 아이템을 구매하거나 무사수행을 통해 골드를 획득하며 최종 엔딩에 도달합니다.

---

## 구조

봇은 3개의 독립 계정으로 운영됩니다.

| 봇 | 역할 | 실행 |
|----|------|------|
| 스케줄 봇 | 턴 행동 제출 / 수치 변화 처리 | `npm run schedule` |
| 상가 봇 | 아이템 구매·판매·장착 / 골드 관리 | `npm run shop` |
| 무사수행 봇 | 던전 전투 / 레이드 / 결투 | `npm run combat` |

플레이어 수치와 골드, 인벤토리는 Google Sheets에서 실시간으로 관리됩니다.
행동 효과, 아이템 정의, 마물 데이터는 시트에서 직접 편집할 수 있으며 GM이 코드를 수정할 필요가 없습니다.
전투 세션과 턴 히스토리는 로컬 JSON 파일에 저장됩니다.

## 설치

```bash
git clone https://github.com/your-repo/academy-bot.git
cd academy-bot
npm install
cp .env.example .env
```

## 환경 설정

`.env` 파일을 열고 아래 항목을 채웁니다.

```
MASTODON_URL=https://your.instance.social

MASTODON_TOKEN=스케줄봇_액세스토큰
SHOP_BOT_TOKEN=상가봇_액세스토큰

SPREADSHEET_ID=구글_스프레드시트_ID

HISTORY_PATH=./data/history.json
SESSIONS_PATH=./data/sessions.json
RAIDS_PATH=./data/raids.json
DUELS_PATH=./data/duels.json

MAX_TURNS=24
INITIAL_GOLD=500
```

## 마스토돈 액세스 토큰 발급

봇 계정 3개를 각각 생성한 뒤, 각 계정으로 로그인해서 아래 과정을 반복합니다.

1. 설정 → 개발 → 새 애플리케이션
2. 권한: `read` `write` `follow` 체크
3. 발급된 토큰을 `.env`의 해당 항목에 입력

## Google Sheets 연동

### 서비스 계정 생성

1. [Google Cloud Console](https://console.cloud.google.com) 접속
2. 프로젝트 생성 또는 선택
3. API 및 서비스 → Google Sheets API 활성화
4. IAM 및 관리자 → 서비스 계정 → 새 서비스 계정 생성
5. 키 탭 → JSON 키 생성 → 다운로드
6. 다운로드한 파일을 `google-key.json`으로 이름 변경 후 프로젝트 루트에 위치

### 스프레드시트 공유

1. Google Sheets에서 새 스프레드시트 생성
2. `google-key.json` 안의 `client_email` 값을 복사
3. 스프레드시트 공유 → 해당 이메일 추가 → 편집자 권한 부여
4. 스프레드시트 URL의 ID 부분을 `SPREADSHEET_ID`에 입력

URL 예시: `https://docs.google.com/spreadsheets/d/여기가ID/edit`

### 시트 탭 구성

스프레드시트에 아래 탭을 생성하고 1행에 헤더를 입력합니다.

**Players**
```
accountId | name | 지능 | 매력 | 체력 | 감성 | 사회성 | 도덕성 | 야망 | 위험도 | 의존성 | 스트레스 | 평판 | 전투 | 골드 | 턴 | 인벤토리 | 장착
```

**Actions**
```
행동명 | 카테고리 | 최소나이 | 골드 | 지능 | 매력 | 체력 | 감성 | 사회성 | 도덕성 | 야망 | 위험도 | 의존성 | 스트레스 | 평판 | 전투
```

- 카테고리: `교육` `아르바이트` `휴식` `사교` `무사수행`
- 골드: 양수=수입, 음수=지출
- 수치 칸이 비어있으면 0으로 처리됩니다

**Items**
```
아이템명 | 상점 | 가격 | 판매비율 | 슬롯 | 최소나이 | 지능 | 매력 | 체력 | 감성 | 사회성 | 전투 | 평판 | 스트레스 | 위험도 | 설명
```

- 상점: `무기상` `의상실` `잡화점`
- 슬롯: `weapon` `clothing` `none` (none은 즉시 소모)
- 판매비율: 0.5 = 구매가의 50%

**Monsters**
```
마물명 | 장소 | 최소나이 | HP | 공격력 | 방어력 | 골드최소 | 골드최대 | 설명
```

## 실행

봇을 각각 별도의 터미널에서 실행합니다.

```bash
npm run schedule   # 스케줄 봇
npm run shop       # 상가 봇
npm run combat     # 무사수행 봇
```

개발 중 파일 변경 감지 자동 재시작:

```bash
npm run dev:schedule
npm run dev:shop
npm run dev:combat
```

## 커뮤니티 진행 구조

```
1~8턴   초기 성장기 (8~12세)
         수확제 1회차 (8턴 종료 후)

9~16턴  확장 단계 (13~16세)
         수확제 2회차 (16턴 종료 후)

17~24턴 완성 단계 (17~18세)
         위기 발생 → 레이드 → 졸업식 / 엔딩 판정
```

## 명령어 목록

### 스케줄 봇

플레이어 명령

| 명령 | 설명 |
|------|------|
| `[스케줄/행동명] [스케줄/행동명] [스케줄/행동명]` | 이번 턴 행동 3개 제출 (자동 처리) |
| `[상태]` | 내 수치를 단어 조합으로 확인 |

GM 명령

| 명령 | 설명 |
|------|------|
| `[현황]` | 전체 플레이어 진행 상태 |
| `[상세]` | 전체 플레이어 수치 상세 (숨김 수치 포함) |
| `[상세/이름]` | 특정 플레이어 상세 |
| `[강제진행]` | 미제출 플레이어를 강제로 다음 턴으로 진행 |

행동 카테고리

| 카테고리 | 골드 | 개방 나이 |
|---------|------|---------|
| 교육 | 지출 | 나잇대별 개방 |
| 아르바이트 | 수입 | 나잇대별 개방 |
| 휴식 | 없음 | 전 나잇대 |
| 사교 | 상황에 따라 | 전 나잇대 |
| 무사수행 | 전투 결과 | 전 나잇대 |

턴당 동일 행동 2회 이상 선택 시 효율 50% 감소.
무사수행은 턴당 1회만 선택 가능.

---

### 상가 봇

| 명령 | 설명 |
|------|------|
| `[상가/무기상]` | 무기 목록 출력 |
| `[상가/의상실]` | 의상 목록 출력 |
| `[상가/잡화점]` | 잡화 목록 출력 |
| `[구매/아이템명]` | 골드 차감 후 인벤토리에 추가 |
| `[판매/아이템명]` | 인벤토리에서 제거 후 골드 수령 (구매가의 40~50%) |
| `[장착/아이템명]` | 장착 슬롯에 적용 / 잡화는 즉시 소모 |
| `[제거/아이템명]` | 장착 해제 (수치 보정 제거) |
| `[주머니]` | 소지금 및 인벤토리 확인 |

GM 명령

| 명령 | 설명 |
|------|------|
| `[골드지급/이름/금액]` | 특정 플레이어에게 골드 지급 |
| `[골드차감/이름/금액]` | 특정 플레이어의 골드 차감 |

장착 슬롯은 `weapon` (무기), `clothing` (의상) 두 종류입니다.
같은 슬롯 아이템을 장착하면 기존 아이템이 자동 해제됩니다.
장착 중인 아이템은 판매할 수 없습니다.

---

### 무사수행 봇

던전 (DM)

| 명령 | 설명 |
|------|------|
| `[장소/장소명]` | 장소 선택 후 마물과 전투 시작 |
| `[공격]` | 마물 공격 |
| `[탐색]` | 골드 또는 아이템 탐색 (실패 시 기습 피해) |
| `[도망]` | 전투 이탈 시도 (60% 성공) |

스케줄에서 무사수행을 선택하면 봇이 DM을 먼저 발신합니다.

레이드 (퍼블릭)

| 명령 | 설명 |
|------|------|
| `[레이드/보스명]` | 레이드 개설 (GM 전용) |
| `[참가]` | 레이드 참가 신청 |
| `[레이드시작]` | 전투 시작 (GM 전용) |
| `[공격]` | 보스 공격 |
| `[레이드종료]` | 레이드 강제 종료 (GM 전용) |

레이드 승리 시 참가자 전원에게 보상 골드를 균등 분배합니다.

결투 (퍼블릭)

| 명령 | 설명 |
|------|------|
| `[결투/상대계정]` | 결투 신청 |
| `[수락]` | 결투 수락 |
| `[공격]` | 교대로 공격 |

## 수치 시스템

공개 수치 (플레이어에게 단어 조합으로 표시)

| 수치 | 낮음 | 중간 | 높음 |
|------|------|------|------|
| 지능 | 무지한 | 총명한 | 천재적인 |
| 매력 | 눈에 띄지 않는 | 친근한 | 전설적인 |
| 체력 | 허약한 | 건강한 | 불굴의 |
| 감성 | 무감각한 | 섬세한 | 예술적인 |
| 사회성 | 고독한 | 사교적인 | 카리스마 넘치는 |

숨겨진 수치 (플레이어에게 비공개, GM만 확인 가능)

`도덕성` `야망` `위험도` `의존성` `스트레스` `평판` `전투`

## 주의사항

- `google-key.json`과 `.env`는 절대 깃에 올리지 마세요. `.gitignore`에 포함되어 있습니다.
- `data/` 폴더도 커밋에서 제외됩니다. 서버 이전 시 별도 백업이 필요합니다.
- Actions, Items, Monsters 시트 데이터는 5분 간격으로 캐시됩니다. 수정 후 즉시 반영하려면 봇을 재시작하세요.
- 봇 3개는 각각 다른 마스토돈 계정을 사용해야 합니다.
